# ? Profile Page Blank Screen - FIXED

## ?? Issue
Profile page was showing completely blank/white screen with no content visible.

---

## ?? Root Cause

The ProfilePageViewModel was setting empty strings for user properties when no user was authenticated:
```csharp
UserFullName = string.Empty;
UserEmail = string.Empty;
UserRole = string.Empty;
UserStatus = string.Empty;
AccountCreatedDate = string.Empty;
```

This caused the entire UI to not render properly because:
1. Empty string bindings can cause Avalonia to skip rendering some elements
2. No visual feedback that the page was loaded
3. Empty date format string caused issues

---

## ? Fix Applied

### File: `PavamanDroneConfigurator.UI/ViewModels/ProfilePageViewModel.cs`

**Changed:**
```csharp
private void LoadUserDetails()
{
    var user = _authSession.CurrentState.User;
    
    _logger.LogInformation("Loading user details. User is null: {IsNull}", user == null);
    
    if (user != null)
    {
        UserFullName = user.FullName;
        UserEmail = user.Email;
        UserRole = user.Role;
        IsAdmin = user.IsAdmin;
        UserStatus = user.IsApproved ? "Approved" : "Pending Approval";
        AccountCreatedDate = user.CreatedAt.ToString("MMMM dd, yyyy 'at' hh:mm tt");

        _logger.LogDebug("Loaded user profile: {Email} ({Role})", user.Email, user.Role);
    }
    else
    {
        // FIXED: Use fallback values instead of empty strings
        UserFullName = "Guest User";
        UserEmail = "Not logged in";
        UserRole = "User";
        UserStatus = "Not authenticated";
        AccountCreatedDate = DateTime.Now.ToString("MMMM dd, yyyy 'at' hh:mm tt");
        IsAdmin = false;
        
        _logger.LogWarning("No user found in auth session");
    }
}
```
