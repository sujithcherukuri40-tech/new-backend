## User Interactions

### Save Flow
1. User modifies parameters
2. `HasUnsavedChanges` flag set to `true`
3. Status shows "Camera parameters loaded." (unchanged)
4. User clicks "Save Parameters"
5. Loading overlay appears
6. Parameters written to flight controller via MAVLink
7. Success: "Camera parameters saved successfully"
8. Failure: "Failed to save some parameters"

### Refresh Flow
1. User clicks "Refresh"
2. Loading overlay appears  
3. Parameters re-downloaded from flight controller
4. UI updates with latest values
5. `HasUnsavedChanges` reset to `false`

### Reset to Defaults Flow
1. User clicks "Reset to Defaults"
2. All fields reset to factory defaults
3. `HasUnsavedChanges` set to `true`
4. Status: "Reset to defaults. Click Save to apply."
5. User must click "Save Parameters" to persist

## Validation Rules

| Field | Min | Max | Default | Error Message |
|-------|-----|-----|---------|---------------|
| Camera Relay Pin | -1 | 50 | -1 | "Value must be between -1 and 50" |
| Camera Servo ON | 1000 | 2000 | 1900 | "PWM value must be between 1000 and 2000" |
| Camera Servo OFF | 1000 | 2000 | 1100 | "PWM value must be between 1000 and 2000" |
| Camera Trigger Distance | 0 | 1000 | 0 | "Value must be between 0 and 1000 meters" |
| Camera Trigger Duration | 0 | 50 | 10 | "Value must be between 0 and 50 deciseconds" |

## Accessibility

### Tooltips (via HintText)
- Every field has descriptive hint text below
- Clear explanation of units and valid ranges
- Guidance on parameter purpose

### Keyboard Navigation
- Tab order: top-to-bottom, left-to-right
- All controls keyboard-accessible
- Enter key submits forms (Save button)

### Screen Reader Support
- Labels properly associated with inputs
- Error messages announced
- Status updates announced
