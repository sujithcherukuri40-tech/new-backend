## Animation Guidelines

### DO NOT USE:
- ? Slide-in animations
- ? Fade-in/fade-out
- ? Pulsing indicators
- ? Blinking states
- ? Rotation spinners
- ? Bouncing elements
- ? Scaling on hover
- ? Transitions on state changes

### ALLOWED:
- ? Instant background color change on hover
- ? Indeterminate progress bar (calm, slow)

**Rationale:** Mission-critical UX requires predictable, stable UI. No distracting motion.

## Implementation Notes

### XAML Structure
- **Styles Section:** All styles defined in `<UserControl.Styles>`
- **No inline styles** in layout (consistency)
- **Class-based styling** (e.g., `Classes="StepCard"`)

### ViewModel Bindings
- **IsConnected:** Controls button `IsEnabled` and status pill color
- **IsResetting / IsRebooting:** Controls progress bar visibility
- **ResetComplete / ResetFailed:** Controls success/error box visibility
- **StatusMessage:** Bound to status message TextBlock
- **LastDroneMessage:** Bound to MAVLink log TextBlock

### Safety Logic
- **Confirmation dialog** required before Step 1 execution
- **Button state management** based on connection and operation state
- **No auto-retry** - user must manually retry on failure
